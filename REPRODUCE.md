# Reproducibility

Everything behind the paper is regenerated by scripts in `src/eval/` (CPU) and notebooks in
`notebooks/` (the GPU language results). Raw numbers are committed in `results/*.json`; figures in
`results/*.svg`.

```bash
bash reproduce_all.sh          # all MAIN CPU experiments (paper figures), at the paper's seed counts
SEEDS=3 bash reproduce_all.sh  # quicker pass
bash reproduce_all.sh exploratory   # also run the illustrative demos
```

## Environment (verified)

- Python 3.11, **torch 2.2.2** (CPU sufficient for all MAIN experiments), numpy 1.26+ or 2.x (a benign
  NumPy 1.x/2.x import warning may print; results are unaffected).
- Dependency ranges in `requirements.txt` / `pyproject.toml`; container image in `docker/`.
- The language results additionally need `transformers>=4.40, peft, accelerate` (installed by the
  Kaggle setup cell) and a single T4 GPU. On Kaggle, `pip uninstall -y torchao` (a peft import guard).

## Releasing for a paper (persistent identifier)

GitHub is not a permanent archive. For submission:

1. Tag the exact commit used for the paper: `git tag -a paper-v1 -m "frozen for submission" && git push --tags`.
2. Enable the **Zenodo–GitHub** integration and publish the release → obtain a DOI.
3. Cite the DOI (not the repo URL) in the manuscript; add the DOI to `CITATION.cff`.
4. Optionally deposit the LLM adapter weights (or exact regeneration steps) alongside.
